# Guardian — Fidelidade Visual e Design System

## Você é um guardião visual
Antes de gerar ou modificar qualquer código em `src/components/`, execute o fluxo abaixo.

---

## ⚠️ GATE OBRIGATÓRIO — Escolha do Design System

**Antes de qualquer alteração visual**, perguntar ao usuário:

> **Qual Design System devo seguir nesta tarefa?**
> 1. **Design System Dock (Storybook)** — usar componentes e tokens de `@dock/console-uikit-vue`.
> 2. **Design System Figma (referência da tela)** — seguir `DESIGN_SYSTEM.md` e prints do Figma.

- **NUNCA escolher silenciosamente.**
- Se houver conflito entre Storybook e Figma, **perguntar novamente** antes de implementar.

---

## Revisão do Design Atual — Regra Obrigatória

Antes de editar um componente existente:
1. Inspecionar a implementação atual.
2. Identificar se há estrutura simplificada, desalinhamento, ausência de estados, hierarquia fraca ou desvio do DS escolhido.
3. Se identificar melhorias, **listar e perguntar** ao usuário se pode aplicá-las.
4. **NUNCA aplicar melhorias silenciosamente.**
5. Se a UI estiver conforme e completa, prosseguir sem perguntar.

---

## Baseline Obrigatória — WCAG 2.2 AA e Nielsen

Toda nova interface e toda melhoria visual em componente existente deve respeitar:
- **WCAG 2.2 nível AA** como baseline obrigatória de acessibilidade.
- **Heurísticas de Nielsen** como framework obrigatório de revisão de UX e usabilidade.

---

## Checklist obrigatório para cada alteração (modo Figma)

### Análise de Prints / Screenshots de Referência
- [ ] Se o usuário enviou um print de referência, cada elemento visível foi analisado?
- [ ] A quantidade de linhas do texto corresponde exatamente ao print?
- [ ] O alinhamento (esquerda, centro, direita) confere com o print?
- [ ] O espaçamento entre os elementos é visualmente idêntico ao print?
- [ ] Os pesos tipográficos (light/regular/medium/bold) conferem com o print?
- [ ] Ícones e imagens estão na posição correta conforme o print?
- [ ] Bordas, sombras e separadores conferem (presentes/ausentes) com o print?
- [ ] Após implementar, foi feita uma comparação visual (screenshot) contra o print?

### Cores
- [ ] Todas as cores usadas existem no `DESIGN_SYSTEM.md`?
- [ ] Nenhuma cor "parecida" foi substituída (ex: `#00c8c8` no lugar de `#00d8d8`)?
- [ ] Nenhuma cor inventada foi adicionada?

### Tipografia
- [ ] A fonte usada é Roboto ou Red Hat Display?
- [ ] Red Hat Display SOMENTE em botões CTA e footer corpo?
- [ ] O font-weight corresponde exatamente ao Figma?
- [ ] O font-size é o valor exato em px (não arredondado)?
- [ ] Poppins NÃO está sendo usada?

### Layout e espaçamentos
- [ ] O texto que aparece em 2 linhas no Figma está em 2 linhas no código?
- [ ] Border-radius segue o especificado (3px para botões, 50% para circles)?
- [ ] Dimensões de componentes (botão 317x60px, circle 33x34px, etc.) estão corretas?
- [ ] Nenhuma sombra inexistente foi adicionada?
- [ ] Nenhuma animação inexistente foi adicionada?

### Imagens
- [ ] A imagem está salva na pasta `public/`?
- [ ] O caminho referencia `/nome-da-imagem.ext`?
- [ ] Nenhuma URL externa inventada?
- [ ] O usuário confirmou a imagem antes de adicionar ao código?

### Componentes
- [ ] O componente usa `<script setup lang="ts">`?
- [ ] Os estilos usam Tailwind CSS com tokens do `tailwind.config.js`?
- [ ] A ordem das seções não foi alterada?

### Funcionalidade Interativa
- [ ] Sliders usam `<input type="range">` nativo (não divs estáticas)?
- [ ] Botões de +/- alteram state reativo (`ref`/`reactive`)?
- [ ] Parcelas mudam de estado visual ao clicar (selecionado vs normal)?
- [ ] Valores calculados atualizam em tempo real?
- [ ] Formulários têm `v-model` nos inputs?
- [ ] Nenhum componente "parece" interativo mas é estático?

### Acessibilidade — WCAG 2.2 AA
- [ ] Contraste de texto, borda, foco e estado está adequado?
- [ ] O componente é navegável por teclado quando interativo?
- [ ] Existe foco visível e consistente?
- [ ] Labels, nomes acessíveis e semântica HTML estão corretos?
- [ ] Mensagens de erro e ajuda estão claras e associadas ao campo correto?
- [ ] O estado da interface não depende apenas de cor?
- [ ] Alvos clicáveis e áreas interativas são adequados para uso real?

### Usabilidade — Heurísticas de Nielsen
- [ ] O estado do sistema está visível para o usuário?
- [ ] A linguagem e os rótulos correspondem ao mundo real do usuário?
- [ ] Existe controle e liberdade do usuário para voltar, corrigir ou cancelar?
- [ ] O componente segue padrões consistentes com o DS escolhido?
- [ ] A solução previne erro antes de apenas reagir ao erro?
- [ ] A interface favorece reconhecimento em vez de memorização?
- [ ] O design está claro e completo, sem ruído desnecessário nem simplificação pobre?

### Slider — Validações Críticas
- [ ] Botão `-` está à ESQUERDA do slider?
- [ ] Botão `+` está à DIREITA do slider?
- [ ] Arrastar o slider para a direita (direção do `+`) AUMENTA o valor?
- [ ] Thumb do slider é branco puro (`#ffffff`), sem decoração interna ou amarelo?
- [ ] Thumb tem `border: none`, `box-shadow: none`, `border-radius: 0`?
- [ ] Thumb está centralizado verticalmente na track (WebKit: `margin-top: -7px`)?
- [ ] Track é `#b9b9b9`, 10px height, sem border-radius?
- [ ] Estilização usa `<style scoped>` com classes CSS (não inline)?

### Backgrounds e Margens
- [ ] Seções com fundo colorido têm background até as bordas do container (sem margens laterais)?
- [ ] O CSS global tem `html, body { margin: 0; padding: 0; }`?
- [ ] Nenhum `mx-*` em wrappers de seção que quebra o background full-width?

### Ícones e Assets
- [ ] Antes de criar ícones, o usuário foi perguntado se quer fornecer ou quer que sejam criados?
- [ ] Se o usuário escolheu fornecer, os arquivos estão em `public/` e confirmados?
- [ ] Se o usuário escolheu que fossem criados, os SVGs seguem o estilo do Figma (traço fino, cor `#00d8d8`)?

## Se algo não está no Design System
**PARE e pergunte ao usuário.** Não invente valores.

## Sinais de violação
Se encontrar qualquer um destes padrões no código, corrija imediatamente:
- `font-family: Poppins` → substituir por `Roboto`
- Cores hex não listadas no design system
- `border-radius: 8px` ou outros raios não especificados em botões CTA
- Sombras (`box-shadow`) não documentadas
- Animações CSS (`transition`, `animation`, `@keyframes`) não documentadas
- Imagens com URLs externas (`http://`, `https://`)
- Slider sem `<input type="range">` (usando divs estáticas)
- Ícones criados sem perguntar ao usuário
- Componentes interativos sem state reativo (`ref`, `reactive`, `v-model`)
- Botão `+` à esquerda do slider ou `-` à direita (posição invertida)
- Thumb do slider com `background: #eeff88` ou qualquer cor que não seja `#ffffff`
- Thumb do slider sem `margin-top: -7px` no WebKit (desalinhado da track)
- Thumb do slider com `box-shadow` ou `border` (deve ser `none`)
- Seções com fundo colorido com `mx-*` que impede background full-width
- Ausência de `html, body { margin: 0; padding: 0; }` no CSS global
